Binary tree

概念

  • 从根结点到某个结点的路径长度称为结点的层数，根结点为第0层
  • 满二叉树无度为1的结点
  • 完全二叉树最多只有最下面两层的结点度数可以小于2，并且最下面一层的结点都集中在该层最左边的连续位置上

性质

  • 完全二叉树中由根结点到各个结点的路径长度总和在具有同样结点个数的二叉树中达到最小
  • 对于扩充二叉树，E为外部路径长度，I为内部路径长度，n为结点个数，有E=I+2n(归纳易证)
  • 任意一棵二叉树，$n_0=n_2+1$(考虑点的入度与结点的关系)
  • 顺序存储是完全二叉树最简单，最节省空间的存储方式

Heap & Priority queue

  • 堆是一种抽象数据结构，在插入或删除元素后自动调整结构维持有序，可用二叉树实现
  • 优先队列是在队列的基础上定义元素优先级，可用堆实现

Huffman Tree

即带权路径长度最短的树，上课划水没听，看了遍书确实很有意思，相应的Huffman编码也很令人感到惊奇，居然还能这么用(○´･д･)ﾉ

Huffman树的构造

  1. 对于给定n个权值$w_0,w_1,…,w_{n-1}$构成的n棵二叉树集合T，使每棵扩充二叉树只具有一个带权为$w_i$的根结点
  2. 构造新的扩充二叉树，在T中找两权值最小树作为新结点的左右子树，新树根结点赋值为左右子树根结点的和
  3. 在T中删除这两棵树，将新扩展的二叉树加入T
  4. 重复(2)(3)，直到T仅含一棵树

Huffman编码

在以字频为权值的基础上所构建的Huffman树，将向左边赋0，向右边赋1，最后所形成对应字母的01编码，容易注意到这种编码为前缀码也是我们所希望得到的

Tree

概念

  • 树形结构中一个结点，至多有一个前驱结点但可以有多个后继结点
  • 处理时将树作为有序树看待，往往把子结点按从左到右的顺序编号
  • 注意度为2的树并不是二叉树，二叉树要求严格区分左右结点
  • 树的相关顺序结构存储包括带双标记的先根次序表示，带度数的后根次序表示以及带双标记层次次序表示

DFS&BFS

  • 将森林以二叉树形式储存，DFS中先根遍历<=>二叉树先序遍历，后根遍历<=>二叉树中序遍历
  • BFS采用队列结构实现

Graph

概念:

  • 不限制前驱和后继数目，反映一种网状关系
  • 对于有向图G任意两点间都是连通的则称G是强连通图，有向图强连通的极大子图称为改有向图的强连通分支
  • 连通图的生成树是指含有该连通图全部顶点的极小连通子图
  • 自由树是不带简单回路的连通图
  • 网络是指带权连通图

Topological Sorting

拓扑排序用以解决先决条件问题，以线性顺序组织多项任务

算法流程

  1. 从有向图中选出一个入度为0的顶点，并输出
  2. 删除该顶点和以它为起点的所有边
  3. 重复(1)(2)至找不出入度为0的点，若最后仍有顶点未被访问则该图含环

Floyd

  • 利用初始邻接矩阵进行迭代，同时初始化一个路径矩阵一同迭代，最后输出时以递归形式输出即可
  • Floyd时间复杂度$O(n^3)$

MST

MST问题主要是为了解决在图中找最小权值连通图的求解，最小生成树介绍的有两种算法，分别是Prim和Kruskal，两者都是典型的贪心准则，在基于MST性质的基础上进行边的添加

MST性质:G=<V,E>，设U是顶点集V的一个非空真子集，若(u,v)∈E，u∈U，v∈V-U，且(u,v)是符合条件的满足权值W(u,v)最小的边，那么一定存在包含(u,v)的最小生成树(反证法易得)
